# OctoAcme — Phase Handoff Checklist

## Purpose
Standardize phase-to-phase handoffs so ownership, dependencies, and readiness are explicit across initiation, planning, execution, release, and retrospective.

Issue reference: this template is part of process improvements tracked in #3.

## How to use
Use this checklist at each lifecycle gate and capture outcomes in the project repository or board.

## Reusable Handoff Template
- From phase:
- To phase:
- Accountable owner:
- Supporting roles:
- Decision date:
- Status: Approved / Blocked / Needs follow-up
- Open risks/dependencies:
- Required follow-up actions (owner + due date):

## Initiation -> Planning
- [ ] One-pager and success metrics are approved
- [ ] Stakeholders and decision makers are documented
- [ ] Initial risks/dependencies have owners
- [ ] Proposed team roles are identified (PM, PdM, engineering, QA, others as needed)

## Planning -> Execution
- [ ] Prioritized backlog includes acceptance criteria and owners
- [ ] Definition of Done and QA approach are documented
- [ ] Milestones and dependency plan are confirmed
- [ ] Escalation path and communication cadence are agreed

## Execution -> Release
- [ ] Scope for release is confirmed and change log is prepared
- [ ] CI, tests, and security checks passed
- [ ] Rollback and incident contact paths are documented
- [ ] Stakeholder and support communication draft is ready

## Release -> Retrospective
- [ ] Post-release verification outcomes are recorded
- [ ] Production issues/incidents are linked and triaged
- [ ] KPI and adoption signals are captured
- [ ] Retrospective owner, date, and participants are confirmed

## Retrospective -> Next Initiation/Planning Cycle
- [ ] Action items are converted into backlog/issues with owners
- [ ] Top process risks are carried forward with mitigations
- [ ] Template/process updates are proposed where needed
- [ ] Outstanding actions are reviewed at the next planning kickoff
